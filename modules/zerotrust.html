10. Zero Trust Architecture (ZTA)

The traditional "castle-and-moat" security model, where everything inside the network is trusted, is obsolete. Zero Trust is a modern security strategy built on the principle of "never trust, always verify." It assumes that threats exist both outside and inside the network, so no user or device is trusted by default.

Pillars of Zero Trust

Access to resources is granted on a per-session basis, and only after a device has passed a strict verification process based on several pillars:

  • Identity: Who is the user? Are they authenticated with strong methods like Multi-Factor Authentication (MFA)?
  • Device: What is the device? Is it a managed corporate laptop? Is its OS patched and its antivirus up-to-date?
  • Network: Where is the request coming from? Is it an unusual location? Is the network itself secure?
  • Application/Workload: Is this application authorized to access this data?
