9. Network Observability

For decades, network monitoring relied on polling devices with protocols like SNMP. Network Observability is a modern evolution that focuses on understanding a system's internal state from its external outputs. Instead of just asking "is the router up?", we ask "what is the router experiencing right now?".

Key Data Sources

  • Streaming Telemetry: Devices push (stream) data continuously to a collector, rather than waiting to be polled. This provides much higher resolution and real-time insights. gNMI (gRPC Network Management Interface) is a popular standard.
  • Flow Data (NetFlow/IPFIX): Routers and switches summarize traffic flows (who talked to whom, for how long, with how much data) and export this data. It's invaluable for performance analysis, security forensics, and capacity planning.
  • Logs & Events: Unstructured data from devices (Syslog) that can indicate configuration changes, errors, and security events.
