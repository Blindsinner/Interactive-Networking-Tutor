4. Routing Protocols & CLI

A router's primary job is to forward packets between different networks. To do this, it maintains a routing tableâ€”a map of known networks and the best path to reach them. Routers build this table in two ways: through static routes (manually entered by an administrator) or through dynamic routing protocols.

  • Static Route: A fixed path. Simple and secure, but doesn't adapt to network changes.
  • Dynamic Protocols (e.g., OSPF, EIGRP): Routers "talk" to each other to learn about the network topology. They use metrics like "cost" or "hop count" to calculate the best path and can automatically re-route around failures.
